# AI 수임료 어드바이저 
### Quotation · Contract · AI Price Advisor

> 행정사 프리랜서가 견적서 작성부터 수임계약서 생성, AI 기반 시장 단가 조회까지 단일 파일로 처리할 수 있는 올인원 업무 관리 도구

---

## 1. 프로젝트 개요

행정사 업무를 시작하는 프리랜서가 실무에서 가장 먼저 부딪히는 문제인 **"단가를 얼마로 받아야 할까?", "견적서와 계약서를 어떻게 만들지?"** 를 해결하기 위한 웹 애플리케이션입니다.

별도 서버 없이 HTML 단일 파일로 동작하며, 브라우저에서 바로 실행할 수 있습니다. 견적서에서 작성한 정보가 수임계약서로 자동 연동되고, AI가 실시간 웹 검색으로 시장 단가를 조회해 단가 표에 자동 반영하는 구조로 설계됐습니다.

- **개발 기간**: 2026년 3월
- **플랫폼**: 브라우저 (서버리스 단일 HTML 파일)
- **대상 사용자**: 행정사 자격 취득 후 개업을 준비하거나 프리랜서로 활동하는 행정사

---

## 2. 해결하려는 문제

### 문제 1 — 단가 기준이 없다
행정사 수임료는 법무사와 달리 공시 기준이 없어 시장마다 천차만별입니다. 초보 행정사는 스스로 단가를 설정할 기준이 없어 시장가보다 낮게 받거나, 반대로 의뢰인과 갈등이 생기는 경우가 잦습니다.

### 문제 2 — 견적서·계약서 양식이 없다
영세 행정사 사무소는 별도 견적서 없이 구두로 수임료를 협의하는 경우가 많습니다. 이 과정에서 수임 내용과 금액이 불명확해지고, 분쟁으로 이어지는 사례가 발생합니다.

### 문제 3 — 계약서 양식의 법적 공백
시중에 유통되는 행정사 수임계약서 예시에는 착수금·중도금·성공보수 구분, 취하 시 환불 기준, 행정심판 업무의 성공보수 약정 위험성 등 실무상 중요한 조항이 빠져 있습니다.

### 문제 4 — 정보 갱신이 안 된다
수임료 시장가는 수시로 변하지만, 한번 설정한 단가표를 업데이트할 체계가 없습니다.

## 3. 주요 기능

### 📋 견적서 생성기
- 출입국·행정심판·비영리단체·서류작성·행정업무·재개발·보상 등 **18개 업무 유형** 카테고리 분류
- 체크박스 **복수 업무 선택** 및 금액 자동 합산
- **범위 단가 토글**: 고정 금액 또는 "50만~150만원" 형식 전환
- 긴급도 3단계 (일반 / 빠른처리 +30% / 당일처리 +50%) 자동 가산
- 착수금·중도금·성공보수 **슬라이더 납입 구조** 설정
- **상담료** 선택 (30분 5만 / 1시간 10만 / 2시간 15만, 수임 시 면제)
- **행정심판 업무 선택 시 성공보수 자동 잠금** — 변호사법 제109조 무효 위험 안내
- 복수 기관·현장 조사·출장·서류수집·부가세 옵션 토글
- 업무 직접 추가 (커스텀 업무명 + 단가 입력)
- 실시간 미리보기 → 🖨 인쇄 / PDF 저장

### 📄 수임계약서 생성기
- 기존 행정사 업무위임계약서 예시(한글 .hwp) 분석 후 **12개 조항으로 보완**
  - 착수금·중도금·성공보수 분할 납입 구조 표 (신규)
  - 취하·철회 시 환불 기준 (착수 전 50% 공제 등) (신규)
  - 행정심판 성공보수 미포함 법적 근거 조항 (신규)
  - 분쟁 해결: 대한상사중재원 → 관할 법원으로 변경
- **📄 계약서로 전환 버튼** — 견적서 데이터(의뢰인, 금액, 납입 구조) 원클릭 자동 이전
- 인쇄/PDF 최적화 CSS (`@media print`)

### 🗂 저장함
- LocalStorage 기반 견적 저장 (최대 60건)
- 카드별 "견적 불러오기" / "계약서 전환" / "삭제"
- 저장 시 예상 소요시간 메모 누적 → 나중에 단가 조정 근거로 활용

### ✦ AI 단가 상담
- Claude API (claude-sonnet-4-20250514) + 웹 서치 연동
- **5개 섹션 구조화 답변 강제**: 시장 단가 현황 → 케이스별 가산 요소 → 추천 단가 → **출처·근거** → 법적 주의사항
- 검색 결과에서 금액 범위 자동 파싱 → **단가 표에 즉시 반영**
- 검색마다 범위가 넓어지는 방향으로 **누적 갱신** (시장 데이터 축적)
- 웹 서치 URL을 파란 출처 박스로 렌더링
- API 키 없이도 오프라인 참고 단가 제공
- 빠른 질문 칩 (음주운전 행정심판, 영업허가, 체류자격 변경 등)

## 5. 개발 과정

### 1) 앞으로 해야 할 것

- [ ] **시간 투입 데이터 분석**: 저장된 견적의 소요시간 메모를 집계해 업무별 시간당 단가를 자동 계산하는 기능
- [ ] **견적→계약서 번호 연동**: 견적 번호(`Q-20250330-001`)가 계약서에도 참조 번호로 자동 표기
- [ ] **세금계산서 발행 메모**: 청구일·지급 기한·연체이자 자동 계산 보조 기능
- [ ] **지역별 단가 프리셋**: 서울/수도권 가산, 지방 기준 등 지역별 프리셋 저장
- [ ] **모바일 최적화**: 현재 좌우 분할 레이아웃을 모바일에서 탭 전환 방식으로 개선
- [ ] **클라우드 동기화**: 현재 LocalStorage 의존 → Google Drive 또는 Notion API 연동으로 기기 간 공유
- [ ] **PDF 직접 생성**: 브라우저 인쇄 대신 jsPDF 등으로 견적서/계약서 PDF 직접 다운로드

### 2) 발전 아이디어

- **AI 계약서 검토**: 작성된 계약서를 Claude에 넘겨 법적 공백이나 불리한 조항을 자동 감지
- **의뢰인 포털**: 의뢰인이 링크를 통해 견적을 확인하고 전자 서명하는 기능
- **업무 진행 트래커**: 수임 건별로 진행 단계(접수→보완→결과)를 관리하는 칸반 보드
- **난이도 자동 분류**: 케이스 설명을 AI가 분석해 상·중·하 난이도를 자동 태그 → 단가 자동 추천
- **수익 대시보드**: 저장된 견적 데이터를 기반으로 월별 예상 매출·업무 유형별 비중 시각화

## 7. 버그 및 해결 과정

### Bug 1 — 행정심판 성공보수 약정의 법적 위험

**발견**: 초기 견적서 툴에 착수금·중도금·성공보수를 단순 슬라이더로 구현했을 때, 행정심판·이의신청 업무에도 성공보수가 설정될 수 있었습니다.

**문제**: 행정사법 제2조 및 변호사법 제109조에 따라, 행정사가 행정심판에서 "사건 결과에 연동한 성공보수" 약정을 체결하면 무효 판결을 받을 수 있다는 법원 판례가 존재합니다.

**해결**: `appeal: true` 플래그를 업무 데이터에 추가하고, 해당 업무 선택 시 성공보수 슬라이더를 자동 비활성화(`disabled`)하고 중도금이 나머지 비율을 자동 보완하도록 처리. 견적서·계약서 미리보기에 법적 고지 배너 자동 출력.

---

### Bug 2 — HWP 파일 파싱 실패

**발견**: 기존 행정사 업무위임계약서(.hwp)를 분석해 개선된 계약서를 만들려 했으나, LibreOffice가 HWP 파일을 DOCX로 변환하지 못함.

**원인**: HWP 5.0 포맷은 LibreOffice에서 제한적으로 지원되며 파일 내용이 표 형태 이미지로 저장되어 있어 직접 변환이 불가능했습니다.

**해결**: `pyhwp` 라이브러리를 설치해 HTML로 변환 후 BeautifulSoup으로 텍스트를 추출, 조항 내용을 수동으로 파악해 계약서에 반영했습니다.

```bash
pip install pyhwp
hwp5html --output hwp_out contract.hwp
python3 -c "from bs4 import BeautifulSoup; ..."
```

---

### Bug 3 — 단가 자동 파싱 정밀도

**발견**: AI 답변에서 금액 범위를 파싱할 때, "330만~550만원(변호사 소송 기준)"처럼 **비교 대상** 금액까지 잘못 파싱해 단가가 지나치게 높게 갱신되는 현상.

**원인**: 정규식이 컨텍스트 없이 숫자 범위만 추출하다 보니, 행정사 단가가 아닌 참고 비교값도 매칭됨.

**해결**: 파싱 범위를 **만원 단위 패턴만** 추출하도록 제한하고, 기존 단가 대비 **넓어지는 방향으로만 갱신**하도록 로직 변경. 예: 기존 50~150만인데 AI가 30~200만을 제시하면 30~200만으로 확장, 100~120만을 제시하면 기존 유지.

---

### Bug 4 — 계약서 조항 번호 자동 증가

**발견**: 특약 사항 유무에 따라 조항 번호가 달라지는데, 고정 번호를 사용하면 특약 없을 때 "기타" 조항이 제12조가 아닌 제13조로 출력되는 문제.

**해결**: `let artIdx = 1; const an = () => artIdx++;` 패턴으로 호출마다 자동 증가하는 함수형 카운터를 도입해, 특약 조항 존재 여부와 무관하게 번호가 연속으로 이어지도록 처리.

본 도구가 생성하는 견적서·수임계약서는 참고용 초안이며, 법적 효력을 보장하지 않습니다. 실제 사용 전 행정사법 및 관련 법령에 따른 검토를 권장합니다. AI가 제시하는 단가는 시장 참고값이며 특정 수임료를 보장하지 않습니다.
